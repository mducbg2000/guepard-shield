## 2. Data Processing Layer

### DongTing loader (`data_processing/dongting.py`)

- Parse `Baseline.csv` để lấy metadata + splits (DTDS-train/validation/test)
- Load `.log` files → list of syscall names per sequence
- Return Polars DataFrame: `[filename, syscalls: list[str], label, split, kernel_version, syscall_count]`

### LID-DS-2021 loader (`data_processing/lidds2021.py`)

- Traverse scenario dirs → training/validation/test splits
- Parse `.sc` files → extract syscall name column only (Tier 1)
- Parse `.json` → exploit label + timestamps
- Return Polars DataFrame: `[scenario, recording_name, syscalls: list[str], label, split]`

### LID-DS-2019 loader (`data_processing/lidds2019.py`)

- Parse `runs.csv` per scenario cho labels
- Parse `.sc` files → syscall name only
- Return Polars DataFrame: `[scenario, recording_name, syscalls: list[str], label]`

## 3. Feature Engineering Layer

### N-gram extraction (`features/ngram.py`)

- Input: list of syscall names
- Output: sparse frequency vector (unigram, bigram, trigram configurable)
- Build vocabulary từ training set, transform sang fixed-size vector
- API: `NgramVectorizer` class với `fit(sequences)` / `transform(sequences)` interface (sklearn-compatible)

### Windowing (`features/windowing.py`)

- Placeholder module — sliding window với configurable `window_size` và `stride`
- Sẽ implement đầy đủ ở P2 cho Transformer input

## 4. EDA (P1) — Jupytext Notebooks

### `01_eda_dongting.py`

- Distribution: sequence lengths, syscall vocabulary size, class balance
- Top-K syscalls frequency (normal vs abnormal)
- Per-kernel-version sample counts
- N-gram analysis: most discriminative bigrams/trigrams

### `02_eda_lidds2021.py`

- Per-scenario statistics: recording counts, sequence lengths, class balance
- Syscall vocabulary overlap giữa scenarios
- So sánh vocabulary DongTing vs LID-DS-2021

## 5. P1.5 Pilot — DT Distilled vs Direct

### Pipeline (`models/pilot.py` + notebook `03_pilot_dt.py`)

1. **Features:** N-gram vectors từ DongTing training set (dùng `NgramVectorizer`)
2. **RF Teacher:** Train RandomForest trên DongTing train, đo baseline performance
3. **Soft labels:** RF `predict_proba()` → soft probability targets
4. **DT Direct:** DecisionTree train trên hard labels (ground truth)
5. **DT Distilled:** DecisionTree train trên RF soft labels (regression mode, fit probabilities)
6. **Compare** trên DongTing test set

### Metrics

| Metric                | Description                      |
| --------------------- | -------------------------------- |
| Accuracy              | vs ground truth                  |
| F1 (macro)            | vs ground truth                  |
| FPR                   | False positive rate              |
| Fidelity              | DT agreement vs RF teacher       |
| Attack-class Fidelity | Fidelity chỉ trên attack samples |
| Rule Complexity       | #leaves × avg_depth              |
| Tree Depth            | max_depth thực tế                |

### Pivot Decision

- DT-distilled Attack-class Fidelity > DT-direct ≥ 3% → KD pathway validated, tiếp P2
- Ngược lại → chuyển focus sang comparative study

## 6. Decisions

- **Features scope:** Syscall name only (Tier 1). Rich features (timestamp, thread_id, args) deferred to P2+.
- **Encoding:** N-gram frequency vectors cho surrogates, sliding windows (placeholder) cho Transformer.
- **P1.5 Teacher:** Random Forest as proxy Teacher (nhanh, không cần GPU, ~94% accuracy ceiling).
- **Notebook format:** Jupytext `.py` với `# %%` cell separators.
- **Package structure:** `src/guepard/{data_processing, features, models}/`.
