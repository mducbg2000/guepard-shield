## Phase 1: Data Pipeline + EDA

### Step 1 — Project scaffolding

- Tạo package structure: `src/guepard/{__init__, data_processing/, features/, models/}`
- Cấu hình `pyproject.toml` để install `guepard` package (editable mode)
- Verify import `from guepard.data_processing import dongting` hoạt động

### Step 2 — DongTing loader

- `data_processing/dongting.py`: parse `Baseline.csv` (Polars), load `.log` files
- Output DataFrame columns: `filename, syscalls, label, split, kernel_version, syscall_count`
- Map splits từ `kcb_seq_class` (DTDS-train/validation/test), labels từ `kcb_seq_lables`
- Unit test: load vài samples, verify schema + counts khớp README (18,966 total)

### Step 3 — LID-DS-2021 loader

- `data_processing/lidds2021.py`: traverse scenario/split dirs, parse `.sc` + `.json`
- Extract syscall name column từ `.sc` (space-separated, cột `syscall_name`)
- Output DataFrame: `scenario, recording_name, syscalls, label, split`
- Label logic: `test/normal_and_attack` + `json.exploit=true` → attack; training/validation → normal
- Unit test: verify per-scenario counts

### Step 4 — LID-DS-2019 loader

- `data_processing/lidds2019.py`: parse `runs.csv` per scenario, load `.sc` files
- Output DataFrame: `scenario, recording_name, syscalls, label`
- Không có pre-split → loader chỉ trả label, split do user tự chia

### Step 5 — N-gram feature extraction

- `features/ngram.py`: class `NgramVectorizer(n_range=(1,3))`
- `fit(sequences)` → build vocab từ training data
- `transform(sequences)` → sparse matrix (scipy CSR hoặc numpy array)
- Sklearn-compatible interface (`fit_transform`, `get_feature_names_out`)

### Step 6 — Windowing placeholder

- `features/windowing.py`: class `SlidingWindow(window_size, stride)` với interface cơ bản
- Chưa cần implement đầy đủ — chỉ cần signature + docstring cho P2

### Step 7 — EDA DongTing

- `notebooks/01_eda_dongting.py` (jupytext `.py`):
  - Class balance (normal vs abnormal)
  - Sequence length distribution (histogram, percentiles)
  - Syscall vocabulary size, top-K frequent syscalls (bar chart, normal vs abnormal)
  - Per-kernel-version sample counts
  - N-gram discriminative analysis: bigrams/trigrams xuất hiện nhiều ở abnormal nhưng ít ở normal

### Step 8 — EDA LID-DS-2021

- `notebooks/02_eda_lidds2021.py`:
  - Per-scenario: recording counts, sequence lengths, attack ratio
  - Syscall vocabulary size per scenario, overlap matrix
  - So sánh vocabulary DongTing vs LID-DS-2021 (Venn/overlap stats)

**Review checkpoint:** EDA results hợp lý, data pipeline stable → tiếp P1.5.

## Phase 1.5: DT Pilot (DongTing)

### Step 9 — RF Teacher

- `models/pilot.py`: train `RandomForestClassifier` trên DongTing train (n-gram features)
- Hyperparameters: `n_estimators=500, max_depth=None` (để RF tự optimize)
- Đo RF performance trên test: Accuracy, F1, FPR → dùng làm performance ceiling
- Lưu `predict_proba()` trên train set → soft labels

### Step 10 — DT Direct vs DT Distilled

- **DT Direct:** `DecisionTreeClassifier` train trên hard labels
- **DT Distilled:** `DecisionTreeRegressor` fit RF soft probabilities, threshold ≥0.5 → predict
- Sweep `max_depth` ∈ [3, 5, 7, 10, 15, None] cho cả hai → chọn depth cho fair comparison
- Đo trên DongTing test: Accuracy, F1, FPR, Fidelity, Attack-class Fidelity, Rule Complexity, Tree Depth

### Step 11 — Pilot notebook + kết luận

- `notebooks/03_pilot_dt.py`:
  - Bảng so sánh DT-Direct vs DT-Distilled tại mỗi max_depth
  - Plot: Fidelity vs Rule Complexity trade-off
  - Pivot decision: Attack-class Fidelity gap ≥ 3% → proceed P2, else → adjust strategy
